# Cross-Section 实现错误记录

## 错误 1：无 epsilon 的 clipped side 全量渲染

**做法**：着色器保留裁剪侧（d < 0）的所有片元，不做距离过滤。

**结果**：模型被裁剪掉的整个半边都变成高亮色，不是截面轮廓。

**教训**：必须用 `abs(d) < epsilon` 做距离过滤，只保留裁剪面附近的片元。

---

## 错误 2：`side: THREE.FrontSide` 或 `BackSide` 单面渲染

**做法**：只渲染 FrontSide 或只渲染 BackSide。

**结果**：截面轮廓只显示约一半——法线朝向相机的面被渲染，背向的被剔除。

**教训**：截面轮廓来自模型表面的片元，其法线方向因位置而异。必须用 `DoubleSide` 才能看到完整轮廓。

---

## 错误 3：在 epsilon 检查里加 `gl_FrontFacing` 条件分支

**做法**：在片段着色器里对 epsilon 内外做不同处理（近平面全渲染，远平面只渲染背面）。

**结果**：逻辑复杂化，本质上等价于简单 epsilon 检查，反而引入 bug。

**教训**：着色器保持简单。复杂分支不仅难调试，且大概率画蛇添足。

---

## 错误 4：把模型材质 `side` 改成 `DoubleSide`

**做法**：为了填充空心，遍历场景把所有模型材质的 `side` 从 `FrontSide` 改成 `DoubleSide`。

**结果**：模型外观改变（背面从外部也可见），且是错误的方式。

**教训**：不应该修改模型的原始材质。填充内部需要单独渲染背面 pass，用 BackSide 材质 + 同样的裁剪平面。

---

## 错误 5：来回切换 `gl.clippingPlanes` 和 `material.clippingPlanes`

**做法**：先从全局裁剪改成逐材质裁剪，没有充分测试就提交。

**结果**：裁剪可能失效，模型不被切开，截面轮廓消失。

**教训**：裁剪平面的设置方式（全局 vs 逐材质）是正交关注点。在着色器逻辑稳定之前，不要同时改裁剪方式。一次只改一件事。

---

## 错误 6：反复删除/恢复 epsilon 检查

**做法**：多次在"有 epsilon"和"无 epsilon"之间切换。

**结果**：浪费时间，重复已经验证过的结论。

**教训**：用户明确给出的核心算法（`abs(d) < epsilon`）是正确方向，不要偏离。遇到问题时先排查其他因素（深度测试、面剔除）而不是否定核心算法。

---

## 错误 7：depthTest 问题没彻底解决就提交

**做法**：用户确认 `depthTest=false` 时轮廓完整，但没有找到正确的 polygonOffset 值就提交了 `depthTest=true` 版本。

**结果**：提交后轮廓又只剩一半。

**教训**：`depthTest=false` 是调试手段，上线需要正确的深度偏移。在用户确认之前，不提交有已知问题的代码。

---

## 错误 8：用标准材质（MeshStandardMaterial）做填充 pass

**做法**：为填充空心创建 `meshStandardMaterial` + `side: THREE.DoubleSide`，依赖全局裁剪 (`gl.clippingPlanes`) 来裁剪。

**结果**：填充面在裁剪侧的片元被全局裁剪裁掉了，无法渲染。

**教训**：填充 pass 必须用 `ShaderMaterial`（自定义着色器），因为只有 ShaderMaterial 不受全局裁剪影响。标准材质会被 `gl.clippingPlanes` 裁剪。

---

## 错误 9：用 `polygonOffset` 试图把填充推到模型后面

**做法**：给填充 pass 加 `polygonOffset`，试图让填充在非切口处被模型遮挡。

**结果**：填充面和模型面不在同一深度（填充在远墙，模型在近墙/边缘），polygonOffset 改变不了这个事实。

**教训**：填充的背面和模型的正面在几何上就是不同位置，深度测试天然区分它们。不需要 polygonOffset。

---

## 错误 10：增大 epsilon 试图替代填充

**做法**：把 epsilon 从 `minDim * 0.03` 增大到 `minDim * 0.5`，想让更厚的片元层覆盖截面内部。

**结果**：保留侧的模型表面也被着色，效果明显错误。

**教训**：epsilon 只控制**轮廓**的粗细。填充截面内部是**另一个独立机制**——渲染裁剪侧的背面。

---

## 错误 11：把轮廓和填充混在一个着色器 pass 里

**做法**：试图用一个 ShaderMaterial 同时做轮廓高亮和内部填充（在着色器里加 `gl_FrontFacing` 分支等）。

**结果**：逻辑纠缠，两头不讨好。

**教训**：轮廓和填充是两个独立 pass：
- **填充**：BackSide + `d < 0`（无 epsilon）→ 渲染裁剪侧背面，填满截面
- **轮廓**：DoubleSide + `abs(d) < epsilon` → 渲染平面附近的完整轮廓

---

## 错误 12：用户明确指出方向后仍然反复偏离

**做法**：用户一开始就给出方案——`abs(d) < 小阈值` 高亮截面 + 保留被裁剪掉的背面填充内部。但我多次偏离：
1. 去掉 epsilon → 整物体变色
2. 加 `gl_FrontFacing` 分支 → 过度设计
3. 改模型材质 `DoubleSide` → 影响外观
4. 换逐材质裁剪 → 裁剪失效
5. 用标准材质做填充 → 被裁剪
6. 增大 epsilon 替代填充 → 错误理解

**结果**：浪费大量时间，用户反复纠正。

**教训**：用户给的方案经过思考，是正确的方向。应该先严格按照方案实现，遇到具体问题时针对性地解决，而不是否定整个方案另起炉灶。

---

## 规则总结

1. **核心算法不可动摇**：`abs(distance) < epsilon` 做轮廓，由用户指定，已验证。
2. **DoubleSide 做轮廓**：单面渲染 = 轮廓残缺。
3. **BackSide 做填充**：渲染裁剪侧背面（`d < 0`，无 epsilon），填满截面内部。
4. **填充用 ShaderMaterial**：不受全局裁剪影响，手动在着色器里判断 `d < 0`。
5. **轮廓用 ShaderMaterial**：不受全局裁剪影响，`abs(d) < epsilon` 判断。
6. **着色器保持简单**：不加不必要的条件分支。
7. **不修改原始材质**：填充和轮廓都是独立的 ShaderMaterial pass。
8. **一次只改一件事**：裁剪方式、着色器、填充策略分开处理。
9. **用户确认前不提交**：尤其是已知有问题的代码。
10. **严格按照用户方案执行，不自行发挥**。
